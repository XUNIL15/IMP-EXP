Je veux créer une application web complète et professionnelle pour moderniser la gestion d'activité d'import/export (gestion de colis, facturation, suivi des dettes).

Objectif : Remplacer la saisie manuelle actuelle qui fatigue mon oncle.

⚠️ CONTRAINTES IMPORTANTES :
- AUCUN EMOJI dans l'interface → utiliser des icônes SVG (Font Awesome ou Heroicons)
- L'ID d'un colis doit être : ID_réelle + Date (format JJMMAA)
  Exemple : colis "A109" reçu le 29/03/2026 → ID = "A109_290326"
  Exemple : colis "B422" reçu le 01/01/2026 → ID = "B422_010126"

FONCTIONNALITÉS ATTENDUES :

1. DASHBOARD PROFESSIONNEL
   - Interface moderne, responsive (Bootstrap 5 ou Tailwind)
   - Graphiques simples (Chart.js) : évolution des colis/jour, revenus, dettes
   - Indicateurs clés (cartes) : total colis aujourd'hui, total kilos, total montant, total dettes impayées
   - Icônes SVG partout (pas d'emojis)

2. GESTION DES ARRIVAGES (CRUD complet)
   - Formulaire pour ajouter une arrivée :
     * Date d'arrivée
     * Nombre total de colis
     * Poids total (kg)
     * Coût total (FCFA ou €)
   - Pour chaque colis, saisie manuelle de :
     * Son ID réel (ex: A109, B422, C001)
     * Le système génère automatiquement : ID_réelle_JJMMAA
   - Modification et suppression possible

3. SAISIE DES COLIS (par arrivage)
   - Pour chaque arrivée, lister les colis un par un
   - Pour chaque colis :
     * ID complet généré automatiquement (affiché mais non modifiable)
     * Type : "Colis individuel" ou "Colis mixte"
     * Poids du colis (kg)
     * Montant du colis (proportionnel ou saisi)
     * Si individuel : un propriétaire
     * Si mixte : plusieurs propriétaires avec leur poids respectif (le système calcule le montant par personne)

4. GESTION DES CLIENTS/PROPRIÉTAIRES
   - Base clients : nom, téléphone, adresse (optionnelle)
   - Sélectionner un client existant ou en créer un nouveau

5. SUIVI DES PAIEMENTS & DETTES
   - Pour chaque colis (ou ligne propriétaire), indiquer :
     * Montant total dû
     * Montant payé immédiatement
     * Solde restant (dette)
   - Interface pour enregistrer des paiements partiels ultérieurs
   - Historique des paiements par client
   - Édition d'un reçu simple (PDF généré via jsPDF)

6. BILAN JOURNALIER (avec export PDF)
   - Page récapitulative par date :
     * Total colis (individuels + mixtes)
     * Total kilos (individuels + mixtes)
     * Total montant (individuels + mixtes)
     * Total encaissé vs total dû
     * Liste des dettes du jour avec clients concernés
   - 🔴 BOUTON "EXPORT PDF" : génère un PDF professionnel du bilan (avec en-tête, tableau, date d'édition)
   - Le PDF doit contenir exactement les mêmes informations que le bilan affiché à l'écran

7. RAPPORTS & EXTRACTION
   - Export Excel/CSV des données (arrivages, colis, dettes)
   - Filtre par date, client, type de colis

STACK TECHNIQUE :
- Frontend : HTML5, CSS3 (Tailwind CSS ou Bootstrap 5), JavaScript (avec Fetch API)
- Backend : PHP 8.x (procédural ou POO simple)
- Base de données : MySQL (via phpMyAdmin)
- Graphiques : Chart.js
- PDF : jsPDF + autoTable (ou DomPDF si plus simple en PHP)
- Icônes : Font Awesome 6 (version gratuite) ou Bootstrap Icons

STRUCTURE DES TABLES (à adapter) :
- clients (id, nom, telephone, adresse, date_creation)
- arrivages (id, date_arrivee, nb_colis_total, poids_total, cout_total)
- colis (id, arrivage_id, code_colis_complet, type, poids, montant)
  * code_colis_complet = ex: "A109_290326"
- colis_proprietaires (id, colis_id, client_id, poids, montant_du, montant_paye, solde)
- paiements (id, client_id, colis_proprietaire_id, montant, date_paiement, mode)

CONTRAINTES VISUELLES ET TECHNIQUES :
- Interface 100% sans emoji (remplacer 😊 par <i class="fas fa-smile"></i>)
- Responsive (mobile friendly)
- Sécurisation basique (protection XSS, injection SQL avec PDO)
- Interface en français
- Dashboard professionnel avec design sobre (couleurs : bleu/gris/blanc)
- Navigation simple (menu latéral ou top bar avec icônes SVG)
